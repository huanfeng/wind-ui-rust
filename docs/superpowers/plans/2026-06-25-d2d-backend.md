# Direct2D 渲染后端 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为不透明大窗口引入 GPU 加速的 Direct2D 渲染后端，在保持 tiny-skia 软路径零回归的前提下，把填充/渐变/阴影/文字光栅迁到 GPU。

**Architecture:** 先抽出后端无关的 `RenderTarget` 接缝（缝①已有的 `Canvas` + 新增 per-frame 目标）与平台层的 `WinRenderBackend` 生命周期接缝（缝②：begin/end/resize/设备丢失）。软后端 `SkiaBackend` 重构自现有 present 逻辑，零行为变化。再在其上新增 `D2DBackend`（D3D11+DXGI swapchain+D2D context），按窗口级 `accelerated` opt-in 选择。文字复用 DirectWrite：measure 共用，D2D 端用 `DrawTextLayout` 自带的 DirectWrite 栈光栅。

**Tech Stack:** Rust 2021、windows-rs 0.62（Direct2D/Direct3D11/DXGI/DirectWrite）、tiny-skia 0.12。

## Global Constraints

- 文字**必须**走 DirectWrite（系统字体缓存、ClearType）；禁止自建字形图集。verbatim 来自 spec §1。
- v1 **只接管不透明大窗**：flip-model `CreateSwapChainForHwnd`，**不引入 DirectComposition**。透明小窗留软渲染。verbatim 来自 spec §2。
- 后端选择为**窗口级显式 opt-in**：`WindowConfig.accelerated`，默认 `false`。verbatim 来自 spec §2/§8。
- 阴影模糊用 **D2D 内置 `ID2D1Effect`**（Shadow/GaussianBlur）。verbatim 来自 spec §2。
- 软路径**零回归**：现有 156+ 单测 + 截图回归必须全过。
- `run_offscreen`（截图/CI）与 RDP（`SM_REMOTESESSION`）**恒走软后端**。verbatim 来自 spec §8/§9。
- 提交信息：conventional commit，**不加** Constraint/Rejected/Scope-risk 等 AI 附加 trailer（用户全局规则）。
- 提交粒度：每个 Task 末尾提交一次。
- 平台限制：本机为 Windows，**macOS 代码改动只能编译验证、无法运行**；涉及 macOS 的步骤须标注。

pub trait RenderTarget {

impl RenderTarget for PixmapTarget<'_> {
    fn make_canvas<'a>(&'a mut self, engine: &'a mut dyn TextEngine) -> Box<dyn Canvas + 'a> {
        Box::new(SkiaCanvas::with_text(self.pixmap, engine, self.scale))

}

- [ ] **Step 1: 定义 trait + SkiaBackend**

在 win32/mod.rs 把现有 `ensure_pixmap` + `paint` 的 pixmap/present 部分搬进 `SkiaBackend`：

```rust
trait WinRenderBackend {
    fn resize(&mut self, w: i32, h: i32);
    fn paint(&mut self, hwnd: HWND, bg: Color, handler: &mut dyn AppHandler);
}

struct SkiaBackend {
    pixmap: Option<Pixmap>,
    buf_w: i32,
    buf_h: i32,
}

impl SkiaBackend {
    fn new() -> Self { Self { pixmap: None, buf_w: 0, buf_h: 0 } }
    fn ensure(&mut self, w: i32, h: i32) {
        let (w, h) = (w.max(1), h.max(1));
        if self.buf_w == w && self.buf_h == h && self.pixmap.is_some() { return; }
        self.pixmap = Some(Pixmap::new(w as u32, h as u32).expect("分配 pixmap 失败"));
        self.buf_w = w; self.buf_h = h;
    }
}

impl WinRenderBackend for SkiaBackend {
    fn resize(&mut self, w: i32, h: i32) { self.ensure(w, h); }
    fn paint(&mut self, hwnd: HWND, bg: Color, handler: &mut dyn AppHandler) {
        // 原 WindowState::paint 的主体搬到这里：GetClientRect→ensure→fill(bg)→
        // PixmapTarget→handler.render→swap_rb→BeginPaint/SetDIBitsToDevice/EndPaint。
        // 0×0 客户区仍配对 BeginPaint/EndPaint 但不绘制。
        // （完整代码为现有 paint() 主体的等价搬迁，仅 pixmap 字段来源改为 self。）
    }
}
```

### Task 5: `D2DBackend` 初始化 + 清屏呈现

新建 `d2d.rs`，建立 D3D11 device → DXGI swapchain → D2D device context 链路，`paint` 暂只 `Clear(bg)` 后 `Present`，**不画内容**。先打通 GPU 呈现。

**Files:**
- Create: `src/platform/win32/d2d.rs`
- Modify: `src/platform/win32/mod.rs`（`mod d2d;`）

**Interfaces:**
- Consumes: `WinRenderBackend`（Task 3）。
- Produces: `pub(super) struct D2DBackend { ... }` impl `WinRenderBackend`；`pub(super) fn try_create(hwnd: HWND, w: i32, h: i32) -> Option<D2DBackend>`（设备创建失败返回 `None` → 调用方落软后端）。

- [ ] **Step 1: 初始化链路**

`d2d.rs` 实现 `try_create`：
- `D3D11CreateDevice`（`D3D_DRIVER_TYPE_HARDWARE`，flag `D3D11_CREATE_DEVICE_BGRA_SUPPORT`；失败返回 `None`）。
- `IDXGIDevice` ← cast；`IDXGIAdapter`→`IDXGIFactory2`；`CreateSwapChainForHwnd`（`DXGI_SWAP_CHAIN_DESC1`：`Format=DXGI_FORMAT_B8G8R8A8_UNORM`，`BufferCount=2`，`SwapEffect=DXGI_SWAP_EFFECT_FLIP_SEQUENTIAL`，`SampleDesc{Count:1,Quality:0}`，`BufferUsage=DXGI_USAGE_RENDER_TARGET_OUTPUT`）。
- `D2D1CreateFactory`(`D2D1_FACTORY_TYPE_SINGLE_THREADED`)→`ID2D1Device`(`from IDXGIDevice`)→`ID2D1DeviceContext`。
- 存 device/swapchain/context/factory 于结构体。

> windows-rs 0.62 的精确签名（`Option<*const ...>`、`Result` 包装）在执行时对编译器逐项核对；本步逻辑已固定，仅 COM 调用形参形态可能微调。

- [ ] **Step 2: 绑定后备缓冲为 target 的辅助**

私有方法 `bind_target(&self)`：`swapchain.GetBuffer::<IDXGISurface>(0)` → `context.CreateBitmapFromDxgiSurface(surface, &props)`（`D2D1_BITMAP_PROPERTIES1`：`bitmapOptions = TARGET | CANNOT_DRAW`，pixelFormat BGRA premultiplied）→ `context.SetTarget(&bitmap)`。

- [ ] **Step 3: impl WinRenderBackend（仅清屏）**

```rust
impl WinRenderBackend for D2DBackend {
    fn resize(&mut self, w: i32, h: i32) {
        // SetTarget(None) 释放对后备缓冲引用 → ResizeBuffers(0,w,h,UNKNOWN,0) → 重新 bind_target
    }
    fn paint(&mut self, _hwnd: HWND, bg: Color, _handler: &mut dyn AppHandler) {
        unsafe {
            self.bind_target();
            self.context.BeginDraw();
            self.context.Clear(Some(&d2d_color(bg)));
            let _ = self.context.EndDraw(None, None);
            let _ = self.swapchain.Present(1, DXGI_PRESENT(0));
        }
    }
}
```

`d2d_color(Color) -> D2D1_COLOR_F`（直通 RGBA→f32/255）。
